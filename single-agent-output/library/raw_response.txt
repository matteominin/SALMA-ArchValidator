\begin{itemize}[leftmargin=*]

    \item \textbf{Post-conditions:} L'utente esegue l'accesso correttamente oppure viene segnalato errore al login.
    \item \textbf{Main Flow:}
    \begin{enumerate}
        \item Utente inserisce e-mail e password.
        \item Utente preme il tasto Login.
        \item Login avviene con successo.
    \end{enumerate}
    \item \textbf{Alternative Flows:}
    \begin{itemize}
        \item 3A. Se identificativo non nel database, compare scritta ``E-mail non presente''.
        \item 3B. Se identificativo esiste ma password sbagliata, compare scritta ``Password errata''.
        \item 3C. Se non c'\`e connessione al database, compare scritta ``Impossibile connettersi con il database''.
    \end{itemize}
\end{itemize}

\paragraph{UC-4: Signup}

\begin{itemize}[leftmargin=*]

    \item \textbf{Pre-conditions:} Disporre di una e-mail; essere connessi al database.

    \item \textbf{Main Flow:}

\subsection{Architecture Extraction}

\subsubsection{Architectural Pattern}

The architecture is explicitly described as:

\begin{itemize}[leftmargin=*]
    \item \textbf{MVC (Model-View-Controller)} for separation of concerns between data, UI, and control logic.
    \item \textbf{Service Layer} between controllers and data access.
    \item \textbf{DAO (Data Access Object)} for persistence abstraction.
    \item \textbf{Singleton} for connection management and main controller.
\end{itemize}

Overall pattern: \textbf{Layered MVC Architecture with Service and DAO layers}.

\subsubsection{Architecture Analysis}

\textbf{Pattern:} Layered MVC with Service and DAO layers.

\textbf{Summary:} The architecture exhibits a clear separation of concerns:

\begin{itemize}[leftmargin=*]
    \item Presentation: JavaFX views and controllers.
    \item Business logic: Service layer (UserService, LibraryUserService, LibraryAdminService, MainService).
    \item Persistence: DAO layer and PostgreSQL database.
    \item Infrastructure: ConnectionManager Singleton.
\end{itemize}

\textbf{Strengths:}

\begin{itemize}[leftmargin=*]
    \item Clear mapping between use cases and services/controllers.
    \item Proper use of design patterns (MVC, DAO, Singleton, Service Layer).
    \item Database schema is explicit and normalized for the domain.
\end{itemize}

\textbf{Areas for Improvement:}

\begin{itemize}[leftmargin=*]
    \item MainService has broad responsibilities (state, data access, coordination); could be decomposed.
    \item Error handling and transaction management strategies are not described at the architectural level.
    \item No explicit description of how concurrency or multiple simultaneous users are handled.
\end{itemize}
